Executive Summary

AeroFlow Violation AI is a working, end-to-end traffic enforcement system that goes from live camera feed to court-ready challan — automatically, with zero manual intervention in between. It detects 7 categories of traffic violations in real time, scores the quality of each evidence frame before saving it, reads the vehicle's number plate using OCR, and auto-generates a structured PDF challan referencing the applicable Karnataka MVA section and fine amount. Simultaneously, it builds a live violation hotspot heatmap that tells traffic supervisors exactly where to deploy constables and when.

The core idea: Most teams will build a violation detector. We built a complete enforcement workflow — from detection to documentation to deployment intelligence. The camera is the starting point, not the ending point.

Problem Statement

Bengaluru handles over 8 million vehicle trips daily across 54 major intersections. The Bengaluru Traffic Police issue approximately 2,800 challans per day — yet violation rates remain persistently high at key bottlenecks like Silk Board, KR Puram, and Hebbal. Three structural gaps drive this:

❌ Current Reality
  • Officers manually spot violations — missing 60–70% of incidents
  • CCTV footage reviewed retrospectively — not actionable in real time
  • Poor-quality camera images get challenged in court — cases dismissed
  • Challan data entry done manually — errors, delays, disputes
  • Constable deployment based on experience, not data
✅ AeroFlow Solution
  • YOLOv8 detects violations in every frame, every second
  • Real-time detection — challan ready within seconds of violation
  • Evidence scorer grades each frame before saving — court-ready only
  • PDF challan auto-generated with MVA section + plate + image
  • Hotspot heatmap tells supervisors where and when to deploy

Three Layers of Differentiation

⭐
Layer 1: Evidence Grading

Every violation frame is scored on plate readability (0–40 pts), vehicle visibility (0–35 pts), and detection clarity (0–25 pts). Only Grade A (≥75) and Grade B (≥50) frames are saved as evidence. Grade C frames are discarded with a reason logged. This is the difference between evidence that holds up in court and evidence that gets dismissed.

📄
Layer 2: Auto-Challan Generation

Each confirmed, graded violation auto-generates a structured PDF challan containing: the evidence image, vehicle class, number plate, violation type, applicable MVA section, fine amount, timestamp, and location. Eliminates 3 manual steps from the current process. Ready to push to Karnataka's e-challan portal (echallan.parivahan.gov.in).

🗺️
Layer 3: Hotspot Intelligence

All violations are geo-tagged and aggregated into an interactive heatmap showing which junctions are most dangerous at which hours. The system generates plain-English constable deployment recommendations: "Deploy at Silk Board during 08:00–09:00 — 47 red-light violations recorded, peak on weekday mornings." Shifts enforcement from reactive to predictive.

Evidence Grading — The Key Innovation

The evidence scoring system is our most distinctive contribution. Every candidate violation frame is evaluated on three independent axes before a single byte is saved to disk:

AxisMax ScoreHow MeasuredWhy It Matters
Plate Readability40 pts OCR confidence × plate length completeness Unreadable plate = no challan possible
Vehicle Visibility35 pts Bbox area ratio + blur score (Laplacian variance) Distant/blurry vehicle = inadmissible in court
Detection Clarity25 pts Model confidence + overlap penalty Ambiguous scene = disputed violation
Result: Only Grade A (≥75/100) and Grade B (50–74/100) frames are saved and used for challan generation. Grade C frames are discarded with the specific reason logged (e.g., "plate unreadable; vehicle too small"). This means every challan our system generates comes with evidence that is designed to withstand a legal challenge.

Scalability & Real-World Deployment

Camera agnostic

The system works with any IP camera or CCTV feed. Each camera instance runs its own violation detector independently. Multiple cameras feed into a shared violation log and hotspot engine.

e-Challan portal integration

The PDF challan output is structured to match the Karnataka e-challan portal fields. API integration with echallan.parivahan.gov.in can be added in one endpoint call — the data structure is already correct.

Model upgrades are plug-in

The system is built modularly. Swapping in a better helmet model, a seatbelt model, or a higher-accuracy YOLO variant (yolov8s, yolov8m) requires changing one line in config.py. No architecture changes.

Edge deployment ready

YOLOv8n runs at 15+ FPS on CPU. For edge devices (Jetson Nano, Raspberry Pi 4), the model can be exported to TensorRT or ONNX format with one ultralytics command.
